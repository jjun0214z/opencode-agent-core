# Skill: debug

## Trigger
`/debug <증상 또는 에러>`

## What This Is
버그를 진단하고 최소 범위로 수정한다.
진단 없이 수정하지 않는다. 수정은 원인에만 한정한다.

---

## Phase 0: Reproduce

1. 증상/에러 메시지 정확히 파악
2. 재현 조건 확인:
   - 항상 발생 vs 간헐적
   - 특정 입력/상태 조건
3. 에러 스택트레이스 수집

---

## Phase 1: Locate

top-down으로 원인 추적:
1. 에러 발생 지점 파일:라인 특정
2. 호출 스택 역추적
3. 관련 코드 읽기 (가정하지 않고 실제 코드 확인)
4. 원인 가설 수립 → 코드로 검증

원인을 특정하기 전 수정 금지.

---

## Phase 2: Fix

원칙:
- **최소 수정** — 원인 지점만 고친다
- 증상 억제(suppress) 금지 — 근본 원인 수정
- 수정 범위가 크면 `/plan` 먼저
- 관련 없는 코드 정리 금지

---

## Phase 3: Verify

- [ ] 에러 재현 안 됨
- [ ] 기존 동작 영향 없음
- [ ] 타입 에러 없음
- [ ] 원인 → 수정 내용 한 줄 요약 제시

---

## Anti-patterns
- 에러 메시지만 보고 코드 읽지 않고 수정 금지
- try-catch로 에러 삼키기 금지
- 버그 수정과 리팩터링 동시 수행 금지
- `git commit` / `git push` 금지
