# Skill: doc

## Trigger
`/doc [path] [--readme | --api | --inline]`

## What This Is
문서를 자동 생성하거나 기존 문서를 최신화한다.
코드를 읽고 실제 동작을 기반으로 작성한다. 코드와 어긋나는 문서는 없는 것보다 나쁘다.

---

## Phase 0: Scope

문서 타입 결정:
| 플래그 | 대상 | 산출물 |
|--------|------|--------|
| `--readme` | 프로젝트 전체 | README.md |
| `--api` | 공개 API/함수 | API 문서 |
| `--inline` | 특정 파일 | JSDoc/TSDoc 주석 |
| (없음) | 자동 판단 | 적절한 타입 선택 |

---

## Phase 1: Read Before Write

1. 대상 코드 전체 읽기
2. 기존 문서 있으면 읽기 (최신화 vs 신규 판단)
3. 실제 동작/인터페이스 파악 (가정 금지)

---

## Phase 2: Write

원칙:
- **코드가 진실** — 주석이나 PR 설명 아닌 실제 코드 기준
- **WHY 중심** — WHAT은 코드가 설명함
- 예제는 실제 동작하는 것만
- 내부 구현 상세 노출 금지 (공개 인터페이스만)

---

## Phase 3: Verify

- [ ] 코드와 문서 일치 확인
- [ ] 예제 코드 실행 가능 여부 확인
- [ ] 기존 문서와 중복/충돌 없음

---

## Anti-patterns
- 코드 읽지 않고 문서 작성 금지
- 동작하지 않는 예제 코드 삽입 금지
- 구현 상세(내부 변수, private 메서드) 문서화 금지
