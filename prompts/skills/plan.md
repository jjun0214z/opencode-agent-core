# Skill: plan

## Trigger
`/plan <topic>`

## What This Is
요구사항을 분석하고 실행 가능한 계획을 생성한다.
구현에 들어가기 전, 설계 결정 사항을 명확히 하고 사용자 확인을 받는다.

---

## Phase 0: Clarify

요청에서 다음을 파악:
- **목표**: 무엇을 만드는가
- **제약**: 기술 스택, 기존 패턴, 타임라인
- **미결 사항**: 결정이 필요한 항목 (2개 이상이면 하나씩만 질문)

---

## Phase 1: Analyze

관련 파일/코드를 읽어 현재 상태 파악:
- 기존 패턴 확인
- 영향 받는 모듈 식별
- 잠재적 충돌 지점

---

## Phase 2: Draft Plan

```markdown
## 계획: {제목}

### 목표
{1-2줄}

### 범위
변경 파일: [목록]
영향 범위: [adjacent modules]

### 단계
1. {단계} — {성공 기준}
2. ...

### 결정 사항
| 결정 | 근거 | 대안 |
|------|------|------|

### 리스크
- {리스크}: {완화 방법}

### 미결 사항 (사용자 확인 필요)
- [ ] {질문}
```

---

## Phase 3: Confirm

계획을 사용자에게 제시하고 확인 받은 후 구현 시작.
확인 없이 코드 작성 금지.

---

## Anti-patterns
- 미결 사항 있는 상태로 구현 돌입 금지
- 코드 읽지 않고 계획 작성 금지
- 한 번에 여러 질문 금지
